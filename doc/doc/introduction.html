Container_bptree

Introduction

1. Is it possible to sum up any N consecutive numbers in less than N-1 operations?

Yes, it is possible, but it is not completely trivial. The best possible efficiency of an algorithm is often limited by the complexity of a data structure. For example, both a linked list and a tree can represent a sequence, but out of these two data structures only a tree can support a search algorithm with logarithmic running time.

In the current version of the namespace container_bptree, one variant of a B+ tree supports an algorithm, which sums up any N consecutive numbers in O(log N) operations in the worst case.

2. Why is it important?

The addition is a fundamental operation in the modern computational world. This operation passes through every level of practical use of computers from single bits to large scale applications, such as databases or models of financial markets. Many applications deal with huge and constantly growing volumes of data and thus require the increasing computational power of systems.

An algorithm, which sums up 1,000,000,000 elements in only 100 operations, is astronomically more cost effective against an equivalent hardware upgrade. It can help improve efficiency and quality of data analysis, problem solving and business modeling.

Advanced data structures offer advanced facilities for data processing and analysis. The B+ trees implemented in this project represent the class of advanced data structures. In addition to super fast summation, these trees provide several other useful extensions of STL containers and iterators.
